Service Worker Modes

Choose how Service Workers are handled for legacy browsers.

Emulation Mode Inactive

Bypass Service Worker registration entirely. The proxy intercepts requests that would normally go through the Service Worker and handles them directly. Simple and reliable.

Remote Mode Inactive

Execute Service Workers in a remote modern browser via WebSocket bridge. The legacy device communicates with a headless browser that runs the actual Service Worker code.

Note: These settings can be changed in the Configuration page.

Service Worker Bundling

When not using remote mode, Service Workers can be bundled and transpiled for legacy browser support.

/__revamp__/sw/bundle?url=<encoded-sw-url>&scope=<encoded-scope>
URL-based Service Worker bundling endpoint
POST /__revamp__/sw/inline
Body: { "code": "<sw-code>", "scope": "/" }
POST Service Worker code directly for transformation
